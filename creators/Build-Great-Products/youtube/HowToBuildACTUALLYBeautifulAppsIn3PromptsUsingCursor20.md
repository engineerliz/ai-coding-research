## Summary

Chris Ashby demonstrates a three-prompt Cursor 2.0 workflow that turns a curated Dribbble screenshot into a reusable design system. Prompt one has GPT-5 analyze the reference image and author a `design.json` style brief, prompt two asks Claude Sonnet 4.5 to scaffold a React + Vite + Tailwind gallery of every component, and prompt three codifies the look-and-feel in `design/design-system.json` so future AI agents inherit the same language. He repeatedly runs the project locally, inspects hover states, and tweaks palette/spacing until Cursor‚Äôs agents and any follow-on workflows can regenerate production-ready UI without falling back to generic gradients.
